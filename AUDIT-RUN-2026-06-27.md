## 2. Findings at a glance (19 total: 0 Critical · 3 High · 3 Medium · 13 Low)

**The story:** the security and SSR-safety posture is genuinely solid (and the adversarial
pass killed several false alarms). The real risk lives in the **`createTactile` instance
lifecycle**, where three High findings are all reachable from ordinary runtime use and all
point at one fix.

### High — fix first (all in `src/index.ts`, all close together)
- **AUDIT-001 — stale-async channel re-enable race.** A `set({sound:false})` (or pack
  swap) issued while a channel's lazy `import()` is in flight gets clobbered when the
  load resolves and unconditionally re-enables the channel. Widest window with sample
  packs (fetch+decode). Same defect in `enableMotion`.
- **AUDIT-002 — `set({events})` drops prior overrides.** It re-merges from the *immutable
  constructor* config, so a second `set({events})` silently reverts the first. A
  lost-update bug in a public method.
- **AUDIT-003 — no `dispose()`.** Each instance leaks 4 capturing window listeners (+ an
  `AudioContext`) for the page lifetime; under React StrictMode / HMR / SPA churn this
  accumulates and can exhaust the browser's `AudioContext` cap → silent audio failure.

> These three share a fix: add a `dispose()` + capture-and-recheck "spec tokens" in the
> async enables + track the shower timers (AUDIT-005). One focused change closes 001/003/005.

### Medium
- **AUDIT-004 — iOS `buzz` is a single tick**, not the "continuous ~2.5s vibration" the
  README/preset advertise (iOS has no Vibration API). Honest-platform documentation gap.
- **AUDIT-005 — uncancelable particle-shower timers** re-spawn at stale `(x,y)`.
- **AUDIT-006 — O(n²) particle collisions** (~125k checks/frame at the 500 cap) — the
  mobile-jank hotspot under sustained showers.

### Low (13)
Whole-burst drop at the particle cap (007), synth `master.gain` clip risk (008),
`backend` type/doc/behavior drift (009), `test()` gap truncates cues (010), unbounded PWM
expansion for custom long low-intensity steps (011), per-frame emoji lookup (012),
`prefer-const` on `let particles` (013), event-override granularity vs README wording
(014), `tactile-site` vendored-`dist` drift (015), cross-repo doc references (016),
Prettier adoption needs a one-time reformat (017), `play(0)`/`play([])` silently cancels
in-flight vibration (018), synth node lifecycle (019, acceptable).

### Adversarially refuted (recorded so they aren't re-raised)
The `ensureCanvas` resize-listener "leak" (the DOM dedups an identical listener
reference), the claim the new ESLint/CI config was broken (it passes), and the
`sideEffects:false` concern (correct). See `docs/code-audit.md` → "Verified sound".

## 4. Notes & one thing for you to do

- **fallow MCP not auto-added.** The harness blocks an agent from writing `.mcp.json`
  (adding an MCP server runs code — a sensible trust gate). To enable the analyzer the
  dalin family uses, paste the 6-line snippet from `docs/TOOLING.md` into a root
  `.mcp.json`. Optional.
- **Prettier is advisory, not enforced.** The source is hand-formatted and not
  Prettier-clean at any print width, so making `format:check` a hard gate needs a one-time
  `npm run format` reformat in its own commit (AUDIT-017). The CI step is `continue-on-error`
  until then. I did **not** reformat your source (that would be a logic-free but huge diff).
- **The `// AUDIT-NNN` comments live in `dist/` too** (tsc preserves comments). They add
  ~1 KB gz to `index.js` (well under budget) and should be removed when each finding is
  fixed.
- **Recommended sequence:** (1) the `dispose()` + spec-token + timer-tracking change
  (001/003/005), (2) README iOS-buzz + override-granularity wording (004/014), (3) the
  particle perf pass (006/007/012). Each is a self-contained follow-up.
